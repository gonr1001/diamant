
\section{Généralités} \label{general}

\subsection{Le système}
\diamant{} est un système interactive déterministe avec des états
(\ang{interactive deterministic system with states}). Il faut
decomposer le système afin de faire \bld{explicite le
comportement} du système. La documentation tabular doit servir à
décrire une décomposition structurée. Le comportement est défini
par des états et des commandes

\subsection{Le modèle de décomposition}
On considère le système comme un environnement où son état change
selon l'arrivé de commandes externes.

La fonction de transition de \diamant{}

\[ {d}: S \times C \rightarrow S
\]


\[ \begin{array}{ll}{d}(s,c)=  & \mbox{état de \diamant{} après avoir re\c{c}u } \\
           ~ &  \mbox{l'évenement commande}~ c \in C ~\mbox{dans l'état}~ s \in S.
             \end{array}
\]


\paragraph{États}

\begin{enumerate}
    \item l'ensemble ${Mode} = \{ n, e\}$\\
    contenant tous les horaires possibles.

    \item l'ensemble .dia \\
    contenant tous les horaires possibles.

\end{enumerate}


\paragraph{Commandes}

\begin{enumerate}
    \item l'ensemble de toutes les action dans les menus.
    \item l'ensemble de toutes les actions dans les boîtes de
    dialogue.
\end{enumerate}
