\section{Avant de commencer}
\subsection{Quoi, de manière précise, a été votre contribution?}
Dans un logiciel de construction d'horaires il est nécessaire de
avoir une structure de données (classe) modélisant la grille
horaire.

Nous proposons trois types de structures de données pour
représenter la grille horaire~:


\begin{enumerate}
\item 1 dimension
 \item 2 dimensions
 \item 3 dimensions
\end{enumerate}

Nous donnons les avantages et les inconvénients de ces trois
structures utilisées dans le cadre d'un logiciel de construction
d'horaires.

\begin{description}
 \item [Quelle question avez-vous répondu?]
 Les avantages et les inconvénients de  trois
structures de données dans le cadre d'un logiciel de construction
d'horaires.

\item [Pourquoi est importante pour le lecteur cette réponse?]
Faire un choix éclaire de la structure.

\item [Quelle est l'extension de cette réponse?] La définition de
de chaque structure, les avantages et les inconvénients de  trois
structures de données dans le cadre d'un logiciel de construction
d'horaires.

\end{description}

\section{Application de la méthodologie}
\subsection{Les préalables}

\begin{description}
\item [Qu'est-ce que vous voulez dire?] Description précise du
résultat que vous voulez transmettre.

Au moins quatre éléments de comparaison.

\begin{itemize}
\item facilité d'expression de contraintes (périodicité)

\item efficacité ¨(indexage)

\item possibilité de paramètre (XML)

\item réutilisation dans d'autres contextes.
\end{itemize}
 \item [Qui va lire
l'article?]Une description précise de l'auditoire, le niveau
technique, la taille, etc. Nous avons ciblé trois types de
lecteurs :
\begin{enumerate}
\item Faire la publication de l'article dans PATAT 2004. Un
auditoire ouvert mais surtout des chercheurs en informatique ou en
recherche opérationnelle qui travaillent dans le domaine de la
conception d'horaires, peut-être avec une bonne connaissance de
mathématiques ou de l'informatique ou de la recherche
opérationnelle, ou les trois ensemble.

\item Les nouveaux développeurs qui se intégreront à l'équipe de
travail \eXit{}. Quelques étudiantes en génie qui pourraient
s'ajouter au groupe \eXit{}, avec un baccalauréat ou une maîtrise
en génie ou en informatique.




\end {enumerate}
Même si les deux groupes de lecteurs semblent différents, le but
de l'article est de montrer les avantages et l'utilisation de la
structure de grille horaire à $n$ dimensions
\end{description}
