La construction d'horaires est un problème qui se présente dans
plusieurs domaines, comme dans les institutions d'enseignement,
dans les centres de santé, dans la gestion de personnel et autres.
En général, il s'agit d'un problème complexe ($NP-$complet). Il
existe de logiciels pour construire des horaires dans tous ces
domaines. Le point commun de ces logiciels, est d'assigner dans
une grille horaire des événements respectant un maximum de
contraintes. Ou min ou max ...

Il existe plusieurs manières de représenter une grille horaire
dans un programme. Nous proposons, dans cet article, une
évaluation de trois structures de données permettant de
représenter une même grille horaire, soit des structures à 1, 2 et
$n$ dimensions. Les critères de comparaison sont~: capacité de
représenter des contraintes, efficacité, facilité d'adaptation et
réutilisation. La capacité de représenter des contraintes est
supérieure si la structure permet de les exprimer simplement et
directement. L'efficacité est meilleure si le nombre d'opérations
pour valider une contrainte est petit. La facilité d'adaptation
est améliore lorsque  la structure possède un représentation
paramétrable et lorsqu'elle est simple. La réutilisation est une
qualité quand la structure peut être utilisé dans plusieurs autres
domaines d'application.

L'objectif de cette comparaison est permettre de faire un choix
éclairé de la structure de la grille horaire dans  le
développement de logiciel d'horaires.
