
%Dans la maintenance de logiciel, un des aspects qui demande plus
%de temps est la recherche des erreurs ou \emph{bugs} qui se
%présentent chez l'utilisateur.

Les bugs les plus coûteux sont ceux qui se produisent lorsque le
logiciel est utilisé par l'utilisateur final. Il s'agit aussi de
bugs compliqués à trouver car l'utilisateur va rapporter le
problème sans indiquer comment il se produit. Dans le meilleur de
cas, il va donner des indications vagues de ce qu'ìl a fait, mais
jamais une séquence suffisament détaillée. Le développeur va
prendre beaucoup de temps avant d'arriver à reproduire les
conditions conduisant au problème.

C'est ce état de choses qui nous a incités à développer l'outil
jReplay destiné à enregistrer le comportement d'un utilisateur
lorsqu'il s'en sert d'une application, le but étant de reproduire
de façon automatique les conditions conduissant au problème avec
l'aide de l'enregistrement effectué. Le principal avantage de
jReplay est de pouvoir répéter le comportement de l'utilisateur
afin de reproduire la séquence conduisant à un problème logiciel
ou \emph{bug}.


%Ainsi, il sera possible de fournir le logiciel à l'utilisateur en
%lui indiquant qu'en cas d'erreur, le programmeur dispose d'outils
%pour trouver le problème plus rapidement.

L'otil jReplay a été conçu pour des applications interactives Java
construites de preference  le \emph{pattern} MVC (Modèle Vue
Contrôleur). Il a été construit en utilisant l'outil Log4j qui
permet de faire la trace d'une application et d'activer ou de
désactiver certains messages en fonction des besoins. jReplay
demande l'insertion de certaines instructions dans des endroits
très précises du code pour générer la trace. En utilisant la trace
comme entrée de jReplay il est possible de re-exécuter
l'application.

Dans cet article nous présentons une déscription détaillé de
jReplay. Comment modifier un logiciel afin d'ajouter les
instructions nécessaires à produire la trace et comment `la
reéxecution est possible. Nous donnons aussi d'autres possibilités
d'utilisation de jReplay.
