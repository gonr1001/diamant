
Dans la maintenance de logiciel, un des aspects qui demande plus
de temps est la recherche des erreurs ou \emph{bugs} qui se
présentent chez l'utilisateur.  C'est ce fait qui nous a incités à
développer l'outil jReplay destiné à enregistrer le comportement
d'un utilisateur dans une application, le but étant de reproduire
de façon automatique le problème avec l'aide de l'enregistrement
effectué. Le principal avantage est de pouvoir répéter le
comportement de l'utilisateur afin de reproduire la séquence
conduisant à un problème logiciel ou \emph{bug}. Ainsi, il sera
possible de fournir le logiciel à l'utilisateur en lui indiquant
qu'en cas d'erreur, le programmeur dispose d'outils pour trouver
le problème plus rapidement.

jReplay a été conçu pour des applications interactives Java
construites avec le \emph{pattern} MVC (Modèle Vue Contrôleur). Il
a été construit en utilisant l'outil Log4j qui permet de faire la
journalisation d'une application et d'activer ou de désactiver
certains messages en fonction des besoins. jReplay demande
l'insertion de certaines instructions dans des endroits très
précises du code pour générer le Journal de trace. Avec ce dernier
il est possible de re-exécuter l'application.
