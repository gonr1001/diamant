\chapter{Planification de \diamant{}}
Ce document est un complément du document DX.mpp qui représente la
planification de \dx{}.
Son objectif est de donner plus d'information sur chaque livrable.


\section{Livrable 1}

Ce livrable sert à vérifier que les fichiers .sig sont corrects.
\dia{} peut bénéficier de ce livrable car si l'on utilise \dx{} pour
vérifier les fichiers à importer, on est sur qu'il n'y a pas d'erreurs ou
des problèmes sur ces fichiers. \dia{} ne fait pas de vérification et
une erreur sur le fichier peut faire planter le logiciel ou pire
construire un horaire avec des erreurs.

\subsection{Caractéristiques externes}
Dans ce livrable les menus \pro{Fichier}, \pro{Preferences} et \pro{Aide} sont fonctionnels.

Dans le menu \pro{Fichier} les sous menus \pro{Définir fichiers à importer}
et \pro{Importer automatiquement}
sont fonctionnels. Le sous menu \pro{Définir fichiers à importer} permet d'afficher une boîte de dialogue afin
de choisir les fichiers à importer\footnote{Il s'agit de fichier à la \saphir{}}; ensuite une autre boîte de dialogue permet
d'indiquer le nom du fichier \pro{.dim} nécessaire à sauvegarder les nom des fichiers.
Le sous menu \pro{Importer automatiquement} permet d'afficher une boîte de dialogue afin
de choisir le nom du fichier \pro{.dim}nécessaire pour faire l'importation.

Dans le cas de boîtes de dialogue, où plusieurs réponses sont possibles,
Il faut traiter tous les cas. Par exemple, buttons \pro{Ok} et
\pro{Annuler}.



Si pendant l'importation une erreur arrive une boîte de dialogue indique le problème
rencontré.
Si l'importation se passe sans erreur une boîte de dialogue indique le succès de
l'importation.

Dans le menu \pro{Preferences} le sous menu \pro{Options L \& F}
est fonctionnel. Une boîte de dialogue est affichée et il est possible de
changer de Look \& Feel de toute l'application.

Dans le menu \pro{Aide} le sous menu \pro{À propos de Diamant 1.5}
est fonctionnel. Une boîte de dialogue est affichée.

\section{Livrable 2}

Tout ce qui était présent dans le livrable 1 continuera à fonctionner.
Dans le cas de boîtes de dialogue, où plusieurs réponses sont possibles,
Tous les cas seront traités.
Ce livrable est simplement un point de contrôle pour montrer que tout avance bien.
Dans ce livrable les lectures et écritures de tous les fichiers manipulés par \dx{}
seront possibles. La fermeture de l'application et des fenêtres sera traitée.
Le traitements associées à la définition de la grille horaire seront aussi disponibles
via une barre d'outils.


\subsection{Caractéristiques externes}
Dans ce livrable les menus \pro{Fichier}, \pro{Fichier Grille Horaire}
sont complets et fonctionnels.

Dans le menu \pro{Fichier} les sous menus \pro{Nouvel horaire},
\pro{Ouvrir horaire},
\pro{Fermer},
\pro{Enregistrer},
\pro{Enregistrer sous},
\pro{Définir fichiers à importer},
\pro{Importer automatiquement},
\pro{Exporter} et
\pro{Quitter}
sont fonctionnels.

Le sous-menu \pro{Nouvel horaire} contient deux sous-sous-menus
(Cycle et Examen).
Dans le deux cas, il s'affiche une boîte de dialogue permettant
de choisir la grille horaire à utiliser pour la construction de l'horaire.

Le sous-menu \pro{Ouvrir horaire}
affiche une boîte de dialogue permettant
de choisir le fichier \pro{.dia} contenant un horaire.

Le sous-menu \pro{Fermer} ferme la grille horaire à l'écran, si nécessaire
il affiche un dialogue demandant s'il faut enregistrer, si c'est le cas,
les dialogue d'enregistrement de fichier suivent.


Le sous-menu \pro{Enregistrer} enregistre sur un fichier l'horaire à l'écran,
si le nom de l'horaire est \pro{Sans titre} un dialogue de sauvegarde
demandant le nom du fichier est affiché. Si le nom du fichier existe
un dialogue demandant s'il faut l'écraser est affiché.

Le sous-menu \pro{Enregistrer sous} enregistre sur un fichier l'horaire à l'écran,
un dialogue de sauvegarde
demandant le nom du fichier est affiché. Si le nom du fichier existe
un dialogue demandant s'il faut l'écraser est affiché.


Les sous menus \pro{Définir fichiers à importer}
et \pro{Importer automatiquement}
sont fonctionnels. Gardent les mêmes caractéristiques du livrable 1.


Le sous-menu \pro{Exporter} affiche un dialogue pour exporter les données,
mais le fichier n'est pas produit. Un dialogue indique la méthode à appeler.

Le sous-menu \pro{Quitter} quitte l'application en fermant
les fenêtres ouvertes et en demandant s'il faut faire les sauvegardes
des fichiers.
Cliquer sur la X de l'application fait la même chose que \pro{Quitter}.



Dans le menu \pro{Fichier Grille Horaire} les sous-menus \pro{Nouvelle grille},
\pro{Ouvrir Grille},
\pro{Fermer},
\pro{Enregistrer} et
\pro{Enregistrer sous}
sont fonctionnels.

de la grille. Il peut définir la taille de la grille en jours et la priorité
des périodes.
Lorsque un sous-menu \pro{Nouvelle grille} ou un sous-menu
\pro{Ouvrir Grille} sont activés l'utilisateur travaille sur la grille
horaire et une barre d'outils est affichée à l'écran.

ouvre une fenêtre avec une grille horaire par default.

pour choisir le fichier contenant la grille horaire (\pro{.dtt},
une fois le fichier choisi, affiche la barre d'outils et
ouvre une fenêtre avec une grille horaire correspondant au fichier.

La barre à outils permet de changer les nombre de jours d'une grille
horaire, de changer le nom associé à une journée
et de changer la priorité d'une période ou d'un ensemble
des périodes.

listes d'activités inclues et exclues. Une activité est soit inclue avec
tous ses groupes et blocs soit elle est exclue.
Exclue veut dire que cette activité (groupes et blocs) ne sera pas à
l'horaire.
Inclue veut dire que tous les groupes et blocs seront à l'horaire.

listes de blocs  figés, placés et non placées. Une activité est soit inclue avec
