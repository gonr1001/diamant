\subsection{Fonctionnalité \ref{examen}~: Examens}
Cette fontionnalité est en réalité un ensemble de fonctionnalités
nécessaires à la construction d'un horaire des examens. Elle est
décrite dans le manuelle utilisateur.

\subsection{Fonctionnalité \ref{enseig}~: Enseignants}
Cette fontionnalité doit servir à assigner à une activité $n$
enseignants. Dans les boîtes de dialogue affectation d'une
activité ou d'un événement il faut offrir la possibilité d'ajouter
$n$ enseignants. On peut ajouter uniquement des enseignants qui
sont connus par \diamant. C'est-à-dire ceux qui sont dans les
fichiers d'importation. La disponibilité des enseignants ajoutés
est à prendre en compte.

à discuter ajout avec de \verb!JComboBox! ou comme ceux de
modifications.

\subsection{Fonctionnalité \ref{groupe}~: Groupes}
Cette fontionnalité doit servir à assurer une compatibilité entre
le nom assigné à un groupe et le nom affiché par \diamant et
retourné avec l'horaire. Il y a des modifications à faire au SIG
et dans \diamant. Au SIG le nom d'un groupe est définit par une
chaîne de 2 \verb!char!s \verb!00! ou \verb!01! à \verb!99!. Cette
information arrive dans le fichier cours et arrive dans le fichier
étudiants dans le cas de la faculté d'administration (afin
d'inscrire des étudiants dans de groupes de soir). Le SIG doit
accepter la même information dans le fichier exportes horaire et
étudiants. Avant les groupes étaient A, B, C \ldots. un seul
\verb!char!.


Dans \diamant nous avons prévu de traiter les groupes comme une
chaîne des \verb!char!s, ainsi pour effectuer cette modification,
on doit simplement vérifier que les données arrivent à la bonne
place et que les automates de reconnaissance des fichiers lisent
les données avec le nouveau format et que le nom d'un groupe
s'affiche correctement par tout. Bien entendu il faut vérifier que
l'exportation est faite avec les bons données et le bon format.

Quelles classes sont à modifier?

afficher sur la grille uniquement les cours de l'ensemble choisi.

\subsection{Fonctionnalité \ref{details}~: Détails}
Détails n'est pas une fontionnalité. Il s'agit de effectuer des
changements mineurs dans le logiciel pour assurer entre autres~:

\begin{enumerate}
    \item Les termes sont utilisées de manière consistante.
    \item Les boîtes de dialogue s'ajustent selon les informations
    à présenter.
    \item Autres ajustements, les utilisateurs vont les indiquer
\end{enumerate}

une grille faite par  Nouvel horaire (cours ou examens).
