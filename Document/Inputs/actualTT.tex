\chapter{Grille horaire actuelle de \diamant{}}

Ce chapitre présente des exemples de grille horaire, décrit leurs
interprétations et l'énumération des différentes informations
requises par \diamant{} pour la construction de la grille horaire.

\section{La grille horaire}

Les fichiers contenant les informations sur la grille horaire par
défaut sont~:

\begin{itemize}
    \item \pro{StandardC.txt}: pour la grille horaire des cours.
    \item \pro{StandardE.txt}: pour la grille horaire des examens.
\end{itemize}

\subsection{Exemple des données de la grille horaire}
Cet exemple est celui du fichier \pro{StandardC.txt}.

\begin{verbatim}
Faculté de génie
Standard pour la grille horaire de cycle

Nombre de jours total du cycle ; 5;
Nombre total de périodes par jours ; 12;

Période ; Priorité ; heure de début ; heure de fin ;

1       ; 0        ; 8h30           ; 9h30         ;
2       ; 0        ; 9h30           ; 10h30        ;
3       ; 0        ; 10h30          ; 11h30        ;\\
4       ; 0        ; 11h30          ; 12h30        ;\\
5       ; 0        ; 13h30          ; 14h30        ;\\
6       ; 0        ; 14h30          ; 15h30        ;\\
7       ; 0        ; 15h30          ; 16h30        ;\\
8       ; 0        ; 16h30          ; 17h30        ;\\
9       ; 0        ; 17h30          ; 18h30        ;\\
10      ; 1        ; 19h00          ; 20h00        ;\\
11      ; 1        ; 20h00          ; 21h00        ;\\
12      ; 1        ; 21h00          ; 22h00        ;}
\end{verbatim}

Signification des données~:

\begin{itemize}
    \item 1ere ligne: une chaîne de caractères, elle donne le nom de la faculté.
    \item 2e ligne: une chaîne de caractères, elle décrit le type du fichier.
    \item 3e ligne: elle est vide.
    \item 4e ligne: elle présente deux informations séparées par un \texttt{;}.
     La première est une chaîne de caractères \pro{Nombre de jours total du cycle}
     tandis que la seconde est un entier qui donne le nombre de jours.
    \item 5e ligne: elle présente deux informations séparées par un \texttt{;}.
    La première est une chaîne de caractères \pro{Nombre total de périodes par jours},
    tandis que la seconde est un entier qui  donne le nombre de périodes par jours.
    \item 6e ligne: elle est vide.
    \item 7e ligne: elle décrit comment les informations concernant \emph{la période, la priorité, l'heure de début et l'heure de fin} sont rangées.
    \item 8e ligne: elle est vide.
    \item de la 9e à la dernière ligne: chaque ligne donne les informations sur une période de la journée. Chaque information est séparée par un \texttt{;} et est décrite comme suit:
        \begin{itemize}
            \item le numéro de la période.
            \item la priorité de la période (0= priorité normale, 1= priorité basse, 2= priorité nulle)
            \item l'heure de début de la période.
            \item l'heure de fin de la période.
        \end{itemize}
\end{itemize}

\section{Limites de cette grille horaire}

Cette grille horaire ne permet d'affecter les resources
que de façon cyclique c'est-à-dire,
pour construire des horaires sur plusieurs cycles
(13 semaines avec une semaine = un cycle),
nous ne construirons que l'horaire sur un cycle en supposant
 que tous les autres cycles sont identiques
 (resources et contraintes identiques dans tous les cycles).

Cette approche aurait été vrai dans tous les cycles
si les contraintes appliquées à certaines resources étaient elles aussi cycliques.

\textbf{Exemple:}

pour la construction d'horaires de cours à l'Université de Sherbrooke,
si la disponibilité d'un enseignant reste inchangée tout au long des
13 cycles (13 semaines) ou encore,
 si un cours doit être donnée par le même enseignant et dans le même local tout au long des 13 cycles,
 alors cette approche, si elle est valable pour un cycle,
 elle l'est pour tous les autres cycles.
 Or, il se trouve que dans certains cas,
 un cours peut être donné par plusieurs enseignants tout au long d'une session.
