\chapter*{Résumé}
Ce document présente comment  des données \diamant{} résidant dans
des fichiers sont mises dans la forme d'objets, afin d'être
traites à l'intérieur de l'application  \diamant{}.

\diamant{} est un logiciel de construction d'horaires, dans lequel
les données de référence sont des fichiers de texte. Ces fichiers
contiennent des enregistrements qui seront ``transformés'' en
objets reconnus par l'application. Ces objets seront
éventuel\-le\-ment modifiés (uniquement certains champs changent
de valeur) suite à des actions de l'utilisateur. Tous les objets
seront sauvegardés par l'application dans un seul fichier texte
XML, ceci ne sera pas complètement vrai dans \dx{}, ainsi
l'application peut continuer un travail en prenant la dernière
sauvegarde faite.

Ce papier présente les détails de formats de fichiers, fichiers originaux et
fichiers produits; avec des indications sur l'utilisation de
champs et les classes concernant cette partie de l'application ainsi que
la correspondance entre champs des enregistrements fichiers et données membres.

Le fait d'uniformiser le traitement de fichiers donne les
avantages suivants :
\begin{itemize}
    \item Le traitement de tous les fichiers est fait en suivant le même \ang{pattern},
    ce qui favorise la modularité,
    la clarté de l'application et le traitement des données de façon uniforme et centralisée;
    \item Réduit la redondance dans le  code, ce qui diminue les efforts de tests et de débogage.
\end{itemize}
