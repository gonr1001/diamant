\chapter*{Résumé}

C'est vrai que une partie importante dans le temps de
développement d'un logiciel est consacré à la validation et
nettoyage des données, ce processus peut devenir très complexe et
très coûteux.

Le logiciel \diamant{} utilise certains fichiers qui à l'heure
actuel sont en enregistrés en format texte, la gestion et
validation de ce type de fichiers est complexe.

Ce document présente comment des données utilisées par le logiciel
\diamant{} qui résident dans des fichiers text sont chargées dans
une base de données relationnelle, dans cette base de données nous
réalisons certains validations d'intégrité des données. Une fois
que les données sont valides, nous pouvons générer des fichiers en
format XML, qui est devenu un standard pour échanger des données.

Avec ce processus, nous pouvons garantir que \diamant{} peut se
concentrer dans les tâches de créer des horaires, sans distraire
son attention dans les activités de validation des données.

Ce papier présente les détails de formats de fichiers, fichiers
originaux et fichiers produits; avec des indications sur
l'utilisation de champs et les classes concernant cette partie de
l'application ainsi que la correspondance entre champs des
enregistrements fichiers et données membres, tout cela pour le cas
précise de \diamant{}.

Mais aussi nous avons comme objectif montrer un modèle de données
et un processus de validation de données assez ouvert et
extensible pour l'utiliser dans autres cas.
